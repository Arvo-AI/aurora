**`server/connectors/loki_connector/client.py`:**

Create a `LokiAPIError(Exception)` class and a `LokiClient` class following the BigPanda/Coroot `requests.Session` pattern. Key specifications:

1. **Module-level constant:** `LOKI_TIMEOUT = 30`

2. **`LokiAPIError(Exception)`** -- custom exception with message string.

3. **`LokiClient.__init__(self, base_url, auth_type="none", token=None, username=None, password=None, tenant_id=None)`:**
   - Strip trailing `/` from `base_url` and store as `self.base_url`
   - Store `self.auth_type = auth_type`
   - Store `self.tenant_id = tenant_id`
   - Create `self._session = requests.Session()`
   - Set `self._session.headers["Accept"] = "application/json"`
   - If `auth_type == "bearer"` and `token`: set `self._session.headers["Authorization"] = f"Bearer {token}"`
   - If `auth_type == "basic"` and `username` and `password`: set `self._session.auth = (username, password)`
   - If `tenant_id`: set `self._session.headers["X-Scope-OrgID"] = tenant_id`

4. **`_request(self, method, path, **kwargs) -> dict`:**
   - Set `kwargs.setdefault("timeout", LOKI_TIMEOUT)`
   - Build `url = f"{self.base_url}{path}"`
   - Wrap in try/except:
     - `requests.HTTPError` -> log and raise `LokiAPIError(str(exc))`
     - `requests.RequestException` -> log and raise `LokiAPIError("Unable to reach Loki")`
   - Call `self._session.request(method, url, **kwargs)`, then `resp.raise_for_status()`, return `resp.json()`
   - Use logger name `"loki_client"` with `[LOKI]` prefix in log messages (matching Grafana's `[GRAFANA]` pattern)

5. **`_request_raw(self, method, path, **kwargs) -> requests.Response`:**
   - Same as `_request` but returns the raw `requests.Response` object without calling `.json()`
   - Used by `test_connection` which needs to check `/ready` (returns "ready" text, not JSON)

6. **`test_connection(self) -> dict`:**
   - Step 1: `GET /ready` via `_request_raw` -- checks Loki is reachable. Expect `resp.text` to contain `"ready"`. If not, raise `LokiAPIError("Loki instance is not ready")`.
   - Step 2: `GET /loki/api/v1/labels` via `_request` -- validates credentials have read access. Store the result.
   - Return `{"ready": True, "labels": data.get("data", [])}` where `data` is the labels response.
   - This implements the CONN-05 two-step validation.

7. **`query_range(self, query, start, end, limit=100, direction="backward", step=None) -> dict`:**
   - Build params dict: `{"query": query, "start": start, "end": end, "limit": limit, "direction": direction}`
   - If `step`: add `"step": step`
   - Return `self._request("GET", "/loki/api/v1/query_range", params=params)`

8. **`query(self, query, limit=100, time=None, direction="backward") -> dict`:**
   - Build params: `{"query": query, "limit": limit, "direction": direction}`
   - If `time`: add `"time": time`
   - Return `self._request("GET", "/loki/api/v1/query", params=params)`

9. **`labels(self, start=None, end=None) -> list`:**
   - Build params dict (only include start/end if provided)
   - Call `self._request("GET", "/loki/api/v1/labels", params=params)`
   - Return `data.get("data", [])`

10. **`label_values(self, label, start=None, end=None) -> list`:**
    - Build params dict (only include start/end if provided)
    - Call `self._request("GET", f"/loki/api/v1/label/{label}/values", params=params)`
    - Return `data.get("data", [])`

11. **`series(self, match, start=None, end=None) -> list`:**
    - Build params: `{"match[]": match}` (match can be a string or list)
    - If `start`: add `"start": start`; if `end`: add `"end": end`
    - Call `self._request("GET", "/loki/api/v1/series", params=params)`
    - Return `data.get("data", [])`

12. **`index_stats(self, query, start, end) -> dict`:**
    - Return `self._request("GET", "/loki/api/v1/index/stats", params={"query": query, "start": start, "end": end})`

13. **`get_rules(self) -> dict`:**
    - Return `self._request("GET", "/loki/api/v1/rules")`

14. **`get_alerts(self) -> dict`:**
    - Return `self._request("GET", "/prometheus/api/v1/alerts")`

```python
"loki_alerts": """
    CREATE TABLE IF NOT EXISTS loki_alerts (
        id SERIAL PRIMARY KEY,
        user_id VARCHAR(255) NOT NULL,
        alert_uid VARCHAR(255),
        alert_title TEXT,
        alert_state VARCHAR(50),
        rule_group TEXT,
        rule_name TEXT,
        labels JSONB,
        annotations JSONB,
        payload JSONB NOT NULL,
        received_at TIMESTAMP NOT NULL,
        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    );

    CREATE INDEX IF NOT EXISTS idx_loki_alerts_user_id ON loki_alerts(user_id, received_at DESC);
    CREATE INDEX IF NOT EXISTS idx_loki_alerts_state ON loki_alerts(alert_state);
    CREATE INDEX IF NOT EXISTS idx_loki_alerts_received_at ON loki_alerts(received_at DESC);
""",
```

Key differences from grafana_alerts:
- Has `rule_group TEXT` (Loki Ruler groups alerts into groups)
- Has `labels JSONB` and `annotations JSONB` (denormalized from payload for faster querying -- Loki alerts are label-heavy)
- Does NOT have `org_id` (Loki uses tenant_id which is stored in credentials, not per-alert)
- Does NOT have `rule_url`, `dashboard_url`, `panel_url` (Grafana-specific fields)

Do NOT modify any other table definition. Only add the new entry.
